# BEGIN SOLUTION

**Answer:**  
`pizza10.merge(notes.reset_index(), on="kind").shape[0]` and  
`notes.merge(pizza10, left_index=True, right_on="kind").shape[0]` (select all that apply).

**Select:**

- `pizza10.merge(notes.reset_index(), on="kind").shape[0]` ‚Äî `reset_index()` puts `kind` in the columns so merging on `"kind"` matches part (a).

- `notes.merge(pizza10, left_index=True, right_on="kind").shape[0]` ‚Äî merge `notes`‚Äôs index (`kind`) to `pizza10`‚Äôs `"kind"` column.

**Do not select:**

- `pizza10.merge(notes, on="kind")` ‚Äî `kind` is the **index** of `notes`, not a column; this merge is not set up correctly.

- `notes.merge(pizza10, left_index=True, right_on="store")` ‚Äî wrong join key.

- `notes.merge(pizza10.reset_index(), on="kind")` ‚Äî after `reset_index()` on `pizza10`, the merge key is not set up the same way as in the working patterns above (per the official key, do not select this).
