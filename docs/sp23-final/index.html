Problem 1.3

True or False: In the code below, Z is guaranteed to evaluate to True.

    x = sun.groupby(["Country", "Year"]).mean().shape[0]
    y = sun.groupby("Country").mean().shape[0]
    z = (x >= y)

  • True

  • False

Answer: True
Let’s us look at each line of code separately:

  • x = sun.groupby(["Country", "Year"]).mean().shape[0]: This line groups the sun DataFrame by both "Country" and "Year", then computes the mean. As a result, each unique combination of "Country" and "Year" will have its own row. For instance, if there are three years of data for a particular country, that will result in three rows for that country in this grouping.

  • y = sun.groupby("Country").mean().shape[0]: When grouping by "Country" alone, each unique country in the sun DataFrame is represented by one row, regardless of the number of years of data associated with that country.

  • z = (x >= y): This comparison checks whether the number of rows produced by grouping by both "Country" and "Year" (which is x) is greater than or equal to the number of rows produced by grouping only by "Country" (which is y).

Given our grouping logic:

  • If every country in the sun DataFrame has data for only a single year, then the number of rows when grouping by both "Country" and "Year" will be equal to the number of rows when grouping by "Country" alone. In this scenario, x will be equal to y.

  • If at least one country in the sun DataFrame has data for more than one year, then there will be more rows when grouping by both "Country" and "Year" compared to grouping by "Country" alone. This means x will be greater than y.

Considering the above scenarios, there’s no situation where the value of x can be less than the value of y. Therefore, z will always evaluate to True.

  • 2

Answer: 12

We know that the area of each bin in the histogram represents the proportion of the data that falls within that bin’s range. In this case, we are looking for the bins 370-390, 390-410, and 410-430. Each bin has a width of 20 and their respective heights are given as 0.002. To find out what proportion of the data falls within a specific bin, you multiply the height of the bin by its width. For each of these bins:

\text{proportion} = \text{height} \cdot \text{width}
\text{proportion} = 0.002 \cdot 20 = 0.04

This indicates that 4% (or 0.04 as a proportion) of the data falls within each of these bins. And the computed proportion (4%) represents the fraction of U.S. cities in the usa DataFrame that have sunshine hours within each bin’s range. Since there are three bins in the range of 370 to 430, and each has 4% of the cities, the total percentage of cities with sunshine hours between 370 and 430 is:

\text{total precentage} = 3 \cdot 4% = 12%

Now let’s look at the code:

  • cond = (usa.get("Jul") >= 370) & (usa.get("Jul") < 430): This line of code creates a Boolean condition that checks which cities in the usa DataFrame have sunshine hours in July between 370 and 430.

  • mystery = 100 * np.count_nonzero(cond) / usa.shape[0]: This line of code calculates the percentage of cities satisfying the condition. np.count_nonzero(cond) counts the number of cities with sunshine hours in the specified range. Dividing this count by the total number of cities in usa (usa.shape[0]) gives the proportion, which is then multiplied by 100 to convert to a percentage.

By interpreting the histogram and using the code provided, we can conclude that both methods are achieving the same goal: determining that 12% of the cities in the usa DataFrame have sunshine hours in July between 370 and 430.

Problem 3.3

Let m be the mean number of sunshine hours in July for all US cities in sun, in standard units. Select the true statement below.

  • m = -1

  • -1 < m < 0

  • m = 0

  • 0 < m < 1

  • m = 1

  • m > 1

Answer: m = 0

When we convert data into standard units, we are essentially standardizing the data. The formula to compute the standard units z for each data point is:

z = \frac{x - \text{mean}}{\text{standard deviation}}

Here, x is an individual data point, and mean and standard deviation are the mean and standard deviation of the entire dataset, respectively.

Now, when we divide this zero by the standard deviation, we still get zero, regardless of the value of the standard deviation (as long as it’s not zero itself).

Thus, the mean of the data after converting to standard units will always be 0.
